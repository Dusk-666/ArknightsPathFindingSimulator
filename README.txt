明日方舟关卡工作台 使用说明

文件位置说明

直接运行的程序：
release\win-unpacked\Arknights Path Visualizer.exe

项目说明文件：
README.txt

源码主目录：
src

测试目录：
tests


一、这是什么

这是一个可视化的关卡编辑与敌人路径预览工具。

你可以用它完成这些事：
1. 画地图
2. 创建多条路线
3. 创建敌人模板
4. 在时间轴上安排刷怪事件
5. 实时播放并查看敌人逐帧运动
6. 查询任意时刻敌人的位置
7. 查看 nextNode、惯性速度、避障向量等调试信息
8. 导入和导出完整 JSON

它不是单纯的 JSON 查看器，而是一个带编辑器界面的单页应用。


二、如何启动

如果你只想直接使用程序，双击下面这个文件：

release\win-unpacked\Arknights Path Visualizer.exe

如果你是开发者，也可以用源码启动：
1. 打开终端进入本文件夹
2. 执行 `npm install`
3. 执行 `npm run dev`


三、先记住 4 个核心概念

1. 地图
地图由格子组成。每个格子有自己的地块类型，比如普通地块、地面不可通行、障碍物等。

2. 路线
路线决定敌人从哪里出生、去哪里、途中是否经过检查点。

3. 敌人模板
模板相当于“敌人种类”。例如普通兵、冲锋手。模板定义速度、转向、身体宽度等参数。

4. 刷怪事件
刷怪事件不是敌人本体，而是“在什么时间，用什么模板，走哪条路线，刷多少只”。
系统会在编译时把事件展开成真正的敌人实例。


四、第一次打开后你会看到什么

程序默认自带一套示例数据，打开后就能直接播放：
1. 有默认地图
2. 有 2 条路线
3. 有 2 种敌人模板
4. 有多条刷怪事件
5. 可以直接点击“播放”看敌人按不同路线移动

建议第一次使用时，先不要急着改数据，先把默认示例完整看一遍。


五、界面总览

顶部导航有 5 个工作页：
1. 实时预览
2. 路线编辑
3. 敌人模板
4. 刷怪编排
5. 工具

推荐工作流：
1. 先在“实时预览”熟悉地图和播放
2. 再去“路线编辑”做路径
3. 再去“敌人模板”做敌人种类
4. 再去“刷怪编排”安排出怪时间
5. 最后回到“实时预览”检查结果


六、最快上手流程

如果你完全没用过，按下面 7 步来：

1. 打开程序，停留在“实时预览”
2. 点击播放，先看默认敌人怎么走
3. 右侧“地图编辑器”里选一个画笔，在地图上点击或拖动涂格
4. 切到“路线编辑”，选中一条路线，试着点“设起点”“设终点”“+ MOVE”
5. 切到“敌人模板”，复制一个模板，改名字和速度
6. 切到“刷怪编排”，点“新建事件”或“在当前帧添加”
7. 回到“实时预览”，点击播放验证结果

只要会这 7 步，你就已经掌握了整个程序最重要的使用方式。


七、实时预览页怎么用

实时预览页是主页面，负责两件事：
1. 播放模拟
2. 顺手编辑地图

你会看到 3 块主要区域：
1. 左边主舞台
显示地图、路线、敌人、调试图层。

2. 右边控制区
有播放控制、地图编辑器、当前属性。

3. 底部轻量时间条
显示当前时间位置，可以快速拖动时间。

常用操作：
1. 点击“播放”开始模拟
2. 点击“暂停”停在当前帧
3. 点击“2x”切换二倍速
4. 点击“单帧后退”或“单帧前进”逐帧看
5. 拖动进度条或底部时间条跳到任意时刻
6. 鼠标悬停敌人查看它的当前信息

适合在这个页面做的事：
1. 看整体效果
2. 快速改地图
3. 边播边验证路线和刷怪安排


八、地图怎么编辑

地图编辑器在“实时预览”页右侧。

1. 修改地图尺寸
右侧“地图编辑器”里有“宽”“高”输入框。
改好后点击“新建空白地图”。

注意：
1. 新建空白地图会重建地图尺寸
2. 原有格子内容会被重置
3. 路线和刷怪事件仍然存在，但如果超出地图范围，系统会自动裁剪或需要你重新调整

2. 选择画笔
目前支持的地块类型包括：
1. 普通地块
2. 地面不可通行
3. 飞行不可通行
4. 障碍物
5. 高代价障碍

3. 涂格
在主舞台地图上：
1. 单击一个格子，设置为当前画笔
2. 按住鼠标拖动，可以连续涂格

4. 查看格子属性
点击某个格子后，右侧“当前属性”会显示：
1. 选中格子
2. 地块类型
3. 对应的一些说明

补充说明：
1. 普通地块：地面和飞行都可通行
2. 地面不可通行：地面单位不能走，飞行单位可以
3. 飞行不可通行：飞行单位不能走，地面单位可以
4. 障碍物：寻路时视作高代价，会强烈绕开
5. 高代价障碍：也是障碍，但代价更高，可用于制造“尽量别走这里”的效果


九、路线编辑页怎么用

路线编辑页专门负责做路径。

页面结构：
1. 左边是地图舞台
2. 右边是路线编辑器和检查点编辑器
3. 右半部分可以滚动，检查点很多时继续往下滑即可

一条路线包含：
1. 起点
2. 终点
3. 若干检查点
4. 路线参数

常用操作：

1. 新建路线
点击“新建路线”。

2. 选择当前路线
在“当前路线”下拉框中切换。

3. 删除路线
选中路线后点“删除”。

4. 改名字
在“名称”输入框直接修改。

5. 设置起点和终点
1. 点击“设起点”，然后在左边地图上点一个格子
2. 点击“设终点”，然后在左边地图上点一个格子

6. 添加检查点
有 3 种方式：
1. 点“+ MOVE”，再去地图上点目标格
2. 点“+ PATROL”，再去地图上点目标格
3. 点“+ WAIT”直接添加等待检查点

7. 编辑检查点
在右侧检查点列表里可以：
1. 选择检查点
2. 改 type
3. 改 target
4. 改等待秒数
5. 上移
6. 下移
7. 删除

8. 快速定位检查点
右侧“快速定位”下拉框可以直接跳到某个检查点。

路线参数解释：
1. `move_mode`
表示路线默认给地面单位还是飞行单位使用。

2. `allowDiagonalMove`
是否允许斜向直连。
如果开启，像 `A1 -> B2` 这种空角会尽量直接走斜线。

3. `visitEveryTileCenter`
要求更严格地经过每个格子中心。

4. `visitEveryNodeCenter`
要求经过节点中心。

5. `visitEveryNodeStably`
要求更稳定地到达节点中心后再离开。

6. `visitEveryCheckPoint`
要求明确到达每个检查点。

7. `ignoreAllButMoveCp`
忽略非移动检查点的部分影响。

给新手的建议：
1. 普通路线先只用 `MOVE` 检查点
2. 不懂时先保持 `allowDiagonalMove` 开启
3. 高级 visit 选项先不要乱开，默认就够用


十、敌人模板页怎么用

敌人模板页是“敌人种类库”。

一个模板表示一类敌人，不是一只敌人。

常用操作：
1. 点击“添加模板”新建
2. 在下拉框里切换当前模板
3. 点“复制”快速做相似模板
4. 点“删除”移除模板

基础参数：
1. `name`
模板名称，会在时间轴和查询结果中显示

2. `attribute_speed`
基础移速

3. `moveMultiplier`
移速倍率

高级参数：
1. `move_mode_override`
是否强制覆盖路线的移动模式

2. `steeringFactor`
转向响应强度

3. `maxSteeringForce`
最大转向/修正力度

4. `halfBodyWidth`
身体半宽，用于碰撞和避障感

5. `footOffset`
脚底相对实体位置的偏移

6. `cursor_offset`
光标参考点偏移

给新手的建议：
1. 先只改 `name`
2. 再改 `attribute_speed`
3. 如果只是想做“快怪”和“慢怪”，先不要碰高级参数


十一、刷怪编排页怎么用

刷怪编排页专门处理时间轴和事件。

页面里有两个重点：
1. 上方时间轴
2. 下方事件编辑器和事件列表

一个刷怪事件包含：
1. 刷怪时间
2. 敌人模板
3. 路线
4. 数量 `count`
5. 间隔帧 `interval_frames`

常用操作：

1. 新建事件
点击“新建事件”。

2. 在当前帧添加事件
先在实时预览停在某一帧，再到编排页点“在当前帧添加”。
这时时间会自动使用当前帧。

3. 编辑事件
在左侧或右侧选中事件后，可以修改：
1. 刷怪时间
2. 敌人模板
3. 路线
4. 数量
5. 间隔帧

4. 删除事件
选中后点击“删除事件”。

5. 点击时间轴跳转
点击时间轴任意位置，当前帧会跳到对应时刻。

6. 点击时间轴标记
会选中对应事件，并跳到该事件时间。

批量刷怪怎么理解：
如果设置：
1. 时间 = 12秒8帧
2. count = 3
3. interval_frames = 10

那么系统会自动展开成 3 只实例，出生时间分别是：
1. 12秒8帧
2. 12秒18帧
3. 12秒28帧

事件只是编排，实例是展开后的结果。


十二、工具页怎么用

工具页主要放高级功能，不干扰日常编辑。

1. 指定时刻查询
输入例如：
1. `2秒8帧`
2. `10秒0帧`

点击“跳转”后，会跳到对应时刻，并显示当时在场敌人的信息。

2. Debug 叠层
可以开关这些图层：
1. `nextNode`
2. `distanceToTarget`
3. `distanceToEnd`
4. `cursorPos`
5. `footPos`
6. 避障向量
7. 惯性速度

适合什么时候用：
1. 想看一条路线到底往哪引导
2. 想检查平整化后有没有变成斜线
3. 想看某只敌人为什么拐弯

3. JSON 导入导出
这是高级模式。

你可以：
1. 点“同步当前数据”把当前编辑器状态写到 JSON 文本框
2. 手动修改 JSON
3. 点“从 JSON 应用”把修改后的内容重新载入程序

建议：
1. 平时优先用可视化编辑
2. 只有做批量修改或备份时再用 JSON


十三、推荐的完整工作流

做一个新关卡时，推荐按这个顺序：

1. 先建地图
在实时预览页右侧设置宽高，点击“新建空白地图”，然后涂地块。

2. 再建路线
去路线编辑页，先做起点和终点，再补检查点。

3. 再建敌人模板
去敌人模板页，先复制已有模板，再调整速度。

4. 再排刷怪
去刷怪编排页，先做关键波次，再细化数量和间隔。

5. 最后回到实时预览验证
观察：
1. 路线是否正确
2. 是否穿模
3. 是否拐弯太硬
4. 时间轴节奏是否正确


十四、主舞台里看到的东西分别是什么

1. 格子标签
例如 A1、B3、K6。
用于精确定位地图、起点、终点和检查点。

2. 路线折线和节点
不同路线会有不同颜色。

3. 敌人实体
会随时间逐帧移动。

4. 调试箭头和辅助线
取决于你是否在工具页里开启了 Debug 叠层。

5. 当前选中格子的高亮
点击地图后会突出显示。


十五、一些常见问题

1. 为什么我改了数据，预览没有马上动？
程序会自动重新编译模拟。如果改动较多，右侧会短暂显示“重新编译中”。

2. 为什么有的敌人不走我想要的路线？
先检查：
1. 该刷怪事件选的是哪条路线
2. 该敌人模板是否覆盖了移动模式
3. 路线的起点、终点、检查点是否设对
4. 地图上是否有不可通行格挡住了它

3. 为什么检查点很多时找起来麻烦？
去路线编辑页右侧使用“快速定位”下拉框。

4. 为什么敌人没有按两段直线走，而是斜着走？
这是正常的。
如果路线允许斜走，系统会在合法情况下尽量直连 `nextNode`，所以会出现斜线移动。

5. 为什么修改地图后路线效果变了？
因为寻路图会重新编译。障碍、阻挡格、高代价障碍都会影响路径。

6. 为什么 JSON 改错后加载失败？
JSON 模式是高级功能。请先确认：
1. 格式合法
2. 路线 ID、模板 ID、事件引用一致
3. 时间格式正确，例如 `5秒0帧`


十六、给新手的实用建议

1. 先用默认示例练手，不要上来就从空白开始
2. 每次只改一类东西，比如先只改地图，确认没问题再改路线
3. 做路线时，先把检查点数量压到最少，能走通后再加细节
4. 做敌人时，先只改速度，别一开始就调高级参数
5. 刷怪编排时，先做单只，再改成批量
6. 出现问题先去工具页看 `nextNode` 和查询结果


十七、你真正需要记住的最短版

只记下面这几句也够用了：

1. 地图在“实时预览”页右边改
2. 路线去“路线编辑”页做
3. 敌人种类去“敌人模板”页做
4. 出怪时间去“刷怪编排”页做
5. 最后回“实时预览”页播放验证
6. 看不懂就去“工具”页查指定时刻和开 Debug
